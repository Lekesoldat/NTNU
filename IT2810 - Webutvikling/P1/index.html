The Chameleon

About

A late evening in 2013 the creative genius, Mr L., witnessed a wild Chameleon change color, and he was hooked since then. As he was unable to change the color of himself, he dedicated his life to Chameleon inspired SVG- and Canvas Art.

Below is L's first ever creations, a true masterpiece: "The HUMAN Chameleon". The two different techniques display two interpretations of what a Chameleon is.

Documentation

Layout

I've used both Flexbox and CSS Grid in this project. I'm rather familiar with Flexbox, so I wanted to use CSS Grid for practice. CSS Grid is usually better than Flexbox when facing a 2D-layout design or in general when a more complex layout is needed. Mine isn't any complex, however I wanted the practice.

My grid holds and structures the main content. The page is divided in three columns, where the outer ones each take up 3 times as much space as the middle one. The about sections spans over all the columns, the svg spans over the first column, the button the next and the canvas the third. Lastly the documentation spans over all columns. I've applied media queries to the page, making the grid re-order the middle row on smaller screens. The SVG, button and Canvas will stack when the screen is too small to display side by side.

Some of the div's holding content has Flexbox applied. This is mainly for alignment as well as easy vertical and horizontal center alignment, but also for responsiveness.

Art

The SVG is created inline in the HTML document, and the Canvas is created in it's respective file. Both graphics utilize colors for background, head and feet defined in the art.js file. The random calculation of color is also done here.

CSS

Interactive-wise, CSS is used to target hovered SVG-elements changing colors and cursor.

SVG

The SVG consists mainly of rectangles and lines. There's also a circle and a square. The color changes is achieved by using SVG-animations. The spinning body is created with SVG-animation-transformations, - a really nice feature with great and simple syntax. Compared to the Canvas-drawing, the SVG one was rather easy to create. Not having to programatically create each shape makes the code a lot easier to read and handle.

Due to the built in animations and transformations, it was also easier to make the code interractive. Being able to refer to whatever SVG object needed in the CSS with simply an ID, is very appreciated.

Canvas

The Canvas artwork, just as the SVG, consists of lines, squares and a circle. The creation of the different shapes are split into functions, as the canvas is being re-drawn continuously.

Two intervals are running in the background, one each 36th millisecond, and one each 1,25th second. The first one makes the body rotate at the same speed as the SVG, which is one full rotation each 10th second. The second interval iterates a variable making the colors of the head and legs change, - the same speed as the SVG.

To make the body spin, the canvas has to be rotated, a new shape has to be drawn, and the canvas has to be rotated back to normal to not de-align the other content. This happens inside one of the loops, and the canvas has to be re-drawn each iteration. If the canvas is not cleared, the shapes will stack on top of each other resulting in the square body gradually drawing out a circle.

Another functionality in the Canvas art is the mouse tracking, - when a page visitor hovers the canvas and gets too close to the Human Chameleons personal space, the mood is set to sad. This is achieved by checking the mouse's x and y coordinates compared to the "personal space".

jQuery

jQuery makes manipulation, animation and especially event handling easier in HTML.

jQuery is here mainly used to select various elements on the page and applying event handling. For instance the backgrounds are changed by selecting parts of the SVG and the entire canvas with jQuery, then applying the new color.

jQuery also offered some neat toggle functionality. With jQuery I selected the documentation section, checked it's state (hidden or not) and toggled which state was fitting. The jQuery slideToggle applied a smooth expanding animation. I also used the library to manipulate the text attribute of the buttons, as well as hiding and showing disco buttons. Lastly jQuery was used to start/stop SVG-animations on click.

Cross-browser testing

I tested my page in both Chrome, FireFox, FireFox Dev. Edition and Safari. I've also resized to simulate different screen sizes, and the result looks good on all screens. I tried my best to force any errors, but I was not successfull.
